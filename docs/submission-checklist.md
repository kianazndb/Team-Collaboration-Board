# Submission Checklist

Use this checklist before you send us your solution. Mark each item as you complete it.

- [ ] All required features from `task_description.md` are implemented or explicitly documented if omitted.  
- [ ] README includes overview, architecture, setup instructions, AI tooling notes, and challenges.  
- [ ] AI usage log (based on `docs/ai-usage-log-template.md`) is filled in and committed.  
- [ ] Screenshots or a short demo video link are included in the README.  
- [ ] Automated scripts/tests (if added) run successfully; commands are documented.  
- [ ] Environment variables and secrets are safely managed (example `.env.example` provided if relevant).  
- [ ] Application builds and runs locally following the documented steps.  
- [ ] Optional deployment link (if provided) loads and matches the latest repository state.  
- [ ] Known issues or trade-offs are listed in the README.  
- [ ] Repository history is clean and demonstrates your workflow (no secrets, no broken commits).
